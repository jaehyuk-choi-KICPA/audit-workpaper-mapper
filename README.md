감사조서 자동화 시스템 (Audit Workpaper Automation)
비상장 회사 외부감사에서 반복적으로 수행되는 조서 작성 작업을 자동화하기 위한 프로젝트입니다. 정산표·분개장·조회서 데이터를 정해진 조서 양식에 매핑·대사하고, 판단과 서술이 필요한 부분만 LLM(Claude API)으로 처리하는 하이브리드 구조로 설계했습니다.


회계 실무(KICPA)를 배경으로, 개발을 배워가며 만들고 있는 프로젝트입니다. 이 공개 저장소에는 처리 로직(코드)만 포함되며, 데이터와 조서 양식은 포함되어 있지 않습니다.


________________


왜 만드는가
비상장 회사 감사는 계정별 감사 절차가 상당 부분 정형화되어 있고 반복적입니다. 현재는 회계사가 정산표의 계정별 증감을 검토해 감사 포인트를 기술하고, 그 수치를 정해진 조서 양식에 수기로 옮깁니다. 이 정형 반복 작업을 자동화해 회계사가 판단·검토에 집중하도록 하는 것이 목표입니다.


________________


설계 원칙
1. 규칙 기반과 LLM의 역할 분리
정형 작업과 판단 작업을 명확히 나눴습니다.


처리 방식
	담당 작업
	규칙 기반 (Python·openpyxl)
	셀 매핑, 합계 계산, 잔액 대사, 만기·금액 기준 분류
	LLM (Claude API)
	비정형 감사포인트의 계정 매핑, Test 서술 생성
	

정형 매핑·대사를 규칙 기반으로 처리해 환각(hallucination)을 차단하고, 판단·서술이 필요한 부분만 LLM에 위임합니다.
2. Workflow 방식 (자율 에이전트 아님)
감사 절차는 결정론적이므로, 처리 순서를 코드에 명시적으로 정의합니다. LLM이 도구를 자율 선택하는 에이전트 구조 대신, 단계가 통제·검증 가능한 워크플로우로 설계했습니다. (감사는 일관성·추적가능성이 중요하므로 의도적으로 이 방향을 택했습니다.)
3. 검증과 사람 검토(Human-in-the-loop)
* 조서 생성 후 합계 대사·누락 여부를 자동 검증합니다.
* 시스템이 생성하는 조서는 최종 완성본이 아니라 초안입니다. 기본적인 수치 매핑과 표준 절차를 조서에 얹어두면, 회계사가 이를 검토하고 추가 절차를 수행해 보완·완성합니다.
* 재분류 등 판단이 걸린 작업은 LLM이 "제안"만 하고, 회계사가 확정합니다. LLM이 감사 판단을 대체하지 않습니다.


________________


처리 흐름
입력: 정산표(잔액·검증기준) + 분개장(거래내역) + 조회서(외부확인)


   │


   ├─ STEP 1. 데이터 추출       정산표=잔액 / 분개장=계정별 거래 / 조회서=대사용


   ├─ STEP 2. 감사포인트 매핑    계정별 증감분석·절차를 정산표에 정렬


   ├─ STEP 3. 조서 초안 생성     [계정유형별 분기] 데이터 매핑 + 합계 자동대사


   ├─ STEP 4. 서술 생성         Test 목적·방법·결론 작성


   └─ STEP 5. 회계사 보완·완성   초안을 검토하고 추가 절차를 수행해 완성


1차 목표는 "정산표·분개장의 수치를 정해진 조서 양식에 옮기는 수작업"의 자동화입니다. 표준 절차와 기본 매핑을 조서에 얹어 초안을 만들고, 회계사가 이를 보완해 완성합니다.
계정 유형별 처리 분기 (routing)
조서 양식(틀) 자체는 표준화되어 있으나, 같은 틀에 데이터를 채우는 계산·매핑 로직은 계정 성격에 따라 다릅니다. 이 차이를 분기로 처리합니다.


* 잔액형 (현금, 차입금): 기초 → 증감 → 기말 잔액 중심으로 채움
* 변동분해형 (유형자산): 거래를 취득·처분·감가상각 등 원인별로 분류해 채움


즉 분기의 핵심은 "어떤 양식을 쓸지 고르기"가 아니라 "계정별로 다른 데이터 채우기 로직"입니다. 처음부터 모든 분기를 구현하지 않고, 계정을 하나씩 추가하며 필요한 분기를 점진적으로 확장합니다.


________________


구현 범위
난이도와 입력 복잡도를 고려해 단계적으로 구현합니다.


* 감사포인트 매핑 (정산표 → 감사 절차 서술)
* 차입금 조서 (조회서 1:1 대사)
* 현금 조서 (조회서 완전성 체크 — 다중 소스 교차 대사)
* 유형자산 조서 (변동분해형 롤포워드)
* 검토 인터페이스


우선 1개 계정 조서를 end-to-end로 완성한 뒤, 동일 패턴으로 확장하는 방식입니다.

기술 스택
* Python 3.11
* openpyxl — 엑셀 입출력·매핑·대사
* anthropic — Claude API (서술 생성)
* python-dotenv — API 키 관리


향후 확장 후보: 합계 검증 게이트 정교화, MCP(완성 기능의 도구화) — 핵심 기능 검증 후 도입 예정.

데이터 정책 (Compliance)
피감회사 데이터와 법인 양식은 모두 보호 대상이므로, 데이터 거버넌스를 설계에 반영했습니다.


* 이 공개 저장소에는 처리 로직(코드)만 포함합니다. 실무 데이터는 물론, 합성 데이터와 조서 양식도 공개 대상에서 제외합니다. (합성 데이터라도 그 양식 구조가 법인 자산에 해당하므로 제외합니다.)
* 데이터·양식 파일은 비공개 폴더에 두며, 버전 관리에서 제외합니다.
* API 키 등 민감 정보는 환경 변수로만 관리하며 저장소에 포함하지 않습니다.
* 실무 적용 버전은 별도 환경에서 승인 범위 내에서만 다룹니다.
